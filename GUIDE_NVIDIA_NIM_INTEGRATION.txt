================================================================================
GUIDE ULTIME : INTÉGRATION DE NVIDIA NIM (NVIDIA INFERENCE MICROSERVICES)
================================================================================

Ce guide explique les principes fondamentaux, l'architecture et les meilleures 
pratiques pour intégrer les modèles d'IA via NVIDIA NIM dans vos applications 
Web, mobiles et solutions techniques d'entreprise.

--------------------------------------------------------------------------------
1. QU'EST-CE QUE NVIDIA NIM ?
--------------------------------------------------------------------------------
NVIDIA NIM est un ensemble de microservices conçus pour accélérer le déploiement 
de modèles d'IA générative. Il fournit des APIs standardisées (souvent compatibles 
avec le format OpenAI) pour :
- Les Grands Modèles de Langage (LLM : Llama, Mistral, Nemotron...)
- Le Multimodal (Vision - Langage)
- La génération d'images (Stable Diffusion, FLUX)
- Le traitement audio (Audio-to-Text, Text-to-Audio)

--------------------------------------------------------------------------------
2. ARCHITECTURE RECOMMANDÉE : LE PROXY BACKEND
--------------------------------------------------------------------------------
NE JAMAIS appeler l'API NVIDIA NIM directement depuis le navigateur (frontend).

Pourquoi ?
- Sécurité : Votre clé API serait exposée à tous les utilisateurs.
- CORS : Les politiques de sécurité des navigateurs bloquent souvent ces appels.
- Flexibilité : Vous pouvez formater les données, gérer les logs et les quotas.

Schéma de flux :
[Utilisateur] <-> [Votre Frontend (React/Vue)] <-> [Votre Backend (Node/Python)] <-> [NVIDIA NIM API]

--------------------------------------------------------------------------------
3. CONFIGURATION ET AUTHENTIFICATION
--------------------------------------------------------------------------------
1. Obtenez votre clé API : Rendez-vous sur https://build.nvidia.com
2. Stockage : Utilisez des variables d'environnement (.env).
   NVIDIA_API_KEY=nvapi-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

--------------------------------------------------------------------------------
4. EXEMPLES D'IMPLÉMENTATION (NODE.JS / EXPRESS)
--------------------------------------------------------------------------------

A. INTÉGRATION CHAT / LLM (AVEC STREAMING)
Le streaming est crucial pour l'UX (affiche le texte pendant qu'il est généré).

```javascript
app.post("/api/chat", async (req, res) => {
  const response = await fetch("https://integrate.api.nvidia.com/v1/chat/completions", {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${process.env.NVIDIA_API_KEY}`,
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      model: "meta/llama-3.1-405b-instruct", // Exemple de modèle
      messages: req.body.messages,
      stream: true
    })
  });

  // Redirection du flux vers le client
  res.setHeader('Content-Type', 'text/event-stream');
  response.body.pipe(res);
});
```

B. GÉNÉRATION D'IMAGES (FLUX / STABLE DIFFUSION)
Attention : Les modèles d'images ne supportent pas tous les mêmes paramètres.

```javascript
app.post("/api/generate-image", async (req, res) => {
  const response = await fetch("https://ai.api.nvidia.com/v1/genai/black-forest-labs/flux.2-klein-4b", {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${process.env.NVIDIA_API_KEY}`,
      "Content-Type": "application/json",
      "Accept": "application/json"
    },
    body: JSON.stringify({
      prompt: req.body.prompt,
      width: 1024,
      height: 1024
    })
  });

  const data = await response.json();
  res.json(data);
});
```

--------------------------------------------------------------------------------
5. GESTION DES RÉPONSES ET FORMATS
--------------------------------------------------------------------------------

IMAGES (Base64) :
L'API renvoie souvent un objet contenant une chaîne Base64. Côté Frontend :
<img src={`data:image/png;base64,${data.data[0].b64_json}`} />

TEXTE (Streaming) :
Les données arrivent par fragments (chunks). Côté client, utilisez un 
`TextDecoder` pour reconstruire le texte en temps réel.

--------------------------------------------------------------------------------
6. ERREURS COMMUNES ET SOLUTIONS
--------------------------------------------------------------------------------

- Erreur 401 (Unauthorized) : Votre clé API est invalide ou a expiré.
- Erreur 422 (Unprocessable Entity) : Paramètres invalides. Vérifiez si vous 
  envoyez `response_format` ou `steps` à un modèle qui ne les accepte pas.
- Erreur 429 (Too Many Requests) : Quota de jetons (tokens) ou de requêtes dépassé.
- Temps de réponse long : Vérifiez la région ou utilisez le streaming.

--------------------------------------------------------------------------------
7. CONSEILS POUR LES SOLUTIONS TECH ROBUSTES
--------------------------------------------------------------------------------

1. System Prompts : Définissez toujours un "rôle" clair pour l'IA (ex: "Tu es un 
   expert technique...") pour obtenir des résultats constants.
2. Gestion des erreurs : Enveloppez toujours vos appels `fetch` dans des blocs 
   `try/catch` et renvoyez des messages d'erreur clairs à l'utilisateur.
3. Cache : Si les requêtes sont répétitives (ex: description de produits), 
   implémentez un cache (Redis) pour économiser vos crédits API.
4. Validation : Côté frontend, validez la longueur du prompt avant l'envoi pour 
   éviter les requêtes inutiles.
